Real-Time Interactivity in 5G Networks

Jerry Chen, Dan Qian, Jason Zhang

Our Approach

We first aim to understand some properties of mobile networking. We built multiple tools and apps to help with our understanding.

    Echo App - A client server packet echoing application that basically start sending packets from a mobile device to the Rain1 server in the DSN lab.
    Ping App - Application that gets the latency between client and server
    Interarrival-Time App - Application that tried to use interarrival time to estimate bandwidth
    Interactive Drawing App - Application that takes commands from the device, sends it to the server, and echoes back to the device before drawing (to test latency from server after device command input)
    Streaming App - Application that generates one way stream of data at some bit rate and then computes the estimated bandwidth at the receiver. Receiver then responds with the available bandwidth and the client adjusts accordingly.

A New Era of Mobile Networking

Next generation technology is focused on bringing computing power and fast networking speeds to smaller, more portable devices. 4G LTE brought fast video streaming speeds at the cost of latency. The high latency cost made real-time applications infeasible since users would see noticable delays, effectively destroying the real-time experience.

5G brings a heavy improvement to mobile networks. It is touted to bring Gigabit download speeds and sub 10ms latency (for reference LTE is around 40~100 ms). This presents an opportunity for real time communication, which requires sub 100 ms end to end latency. While theoretical 5G has amazing benefits, there is still much work to be done before it is fully realized since its still so new. Thus the goal for this project is to investigate real-time interactivity over a 5G network and design a bandwidth estimation protocol to efficiently utilize bandwidth.

Current Progress

We've built the above apps and we are currently iterating on the Interactivity application and the Streaming application to try and get bidirectional communication between client and server devices and make sure the application is stable on mobile devices so that we can begin doing some realistic 5G mobile testing. We also aim to read more literature on real-time bandwidth estimation protocols and try and implement them to see if we can get any improvements.

What we know now...

Through our tests we've learned a lot of interesting things about mobile networking. First, all wirelessly trasmitted packets are sent to a discrete baseband processor on the mobile device. Second, there is a possiblity of packet bunching at the cell towers before being sent to its destination (more evident in the downlink). Third, measurements of LTE are typically around 40ms for Verizon, 70 ms for AT&T, and 100 ms for T-Mobile. These high latency measurements are possibily due to mobile devices peering somewhere else geographically. From our tests we noted that they do all peer around the Tri-State Area.
